# Kibbutznik Codebase Synthesis — Remaining Issues (verified 2026-05-29)

## Executive Summary

The biggest remaining risks are **authorization gaps on the simulation control surface and a launch-day cache stampede**, both verified live against the running code. Anyone on the internet can pause/restart the public demo, switch the LLM backend, and burn the LLM/Ollama budget through unauthenticated `/simulation/*` and `/tkg/semantic_search` endpoints, while the unlocked 30-second `/highlights` cache will fan out hundreds of DB round-trips per concurrent miss and can exhaust the 40-connection pool under an HN spike. Several governance/financial correctness bugs (stale votes from expelled members, permanently-orphaned membership escrow on amend, the global membership cap bypass via invites) are real but lower-blast-radius. Most schema/model `__table_args__` divergences and the redundant email index are genuine cleanups but carry no live exploit. **Fix the simulation-endpoint authz first** — it is the cheapest, highest-leverage change and directly protects launch-day cost and stability.

## P0 — Fix before any traffic spike

### 1. Unauthenticated `/simulation/*` write endpoints (THE ONE TO FIX FIRST)
- **Location:** `agents/simulation_api.py:114-263` (pause, resume, restart, interview, chat, run-round, POST /llm)
- **Evidence (verified):** No route in the file carries `Depends(require_user)` or any guard. POST `/pause`, `/resume`, `/restart`, `/interview`, `/chat`, `/run-round`, and POST `/llm` are all reachable anonymously through the `/kbz/` proxy. `/interview` and `/run-round` invoke real LLM calls; `/restart` wipes the DB.
- **Recommendation:** Guard every POST/DELETE in `simulation_api.py` with `Depends(require_user)` (or the `X-KBZ-Agent-Secret` header). Read-only GETs (`/status`, `/agents`, `/events`, `/agent-stats`, GET `/llm`) can stay public for the viewer. This is a ~10-line change and is the single most important fix.

### 2. `/highlights` cache stampede — no lock around miss-check + rebuild
- **Location:** `src/kbz/routers/highlights.py:37` (plain dict cache), `:121` (miss check), `:223` (`_root_id_of` per-row in build path)
- **Evidence (verified):** `_CACHE` is a bare dict; the build path between the miss-check and `_CACHE` write contains multiple `await`s, so N concurrent requests on a cold/expired cache all rebuild. Each rebuild calls `_root_id_of` (one SELECT per parent hop, up to 64) per proposal across up to 120 rows.
- **Recommendation:** Add a module-level `asyncio.Lock()`, re-check the cache inside the lock, build only if still stale. Also add `Cache-Control: public, max-age=30` to the response and pre-compute `community_id → root_id` in one batch before the loop. Pair with `pool_timeout=5` in `database.py` (see P1) so a miss fails fast instead of queuing 30s.

### 3. `/tkg/semantic_search` unauthenticated — unbounded Ollama embedding calls
- **Location:** `src/kbz/routers/tkg.py:223-226`
- **Evidence (verified):** `require_user` is imported and used on a sibling endpoint (line 298) but **not** on `semantic_search` — its only injected dependency is `db`. Each call triggers an Ollama embedding HTTP call plus a pgvector KNN query with no rate limit.
- **Recommendation:** Add `Depends(require_user)`, or gate it with the existing per-IP limiter if public search is required.

> **Note on the "KBZ_AGENT_API_SECRET not set" finding (raw item #2):** The *code path* is real and correct — when `settings.agent_api_secret == ''`, `enforce_session_matches_body` (`auth_deps.py:157-158`) falls back to permissive behavior, allowing a cookieless caller to name any `user_id`. Whether the secret is actually set in production is an **ops/env fact not verifiable from the repo**. Treat as **P0 ops checklist** (set `KBZ_AGENT_API_SECRET`, restart, strip the header at nginx) rather than a code change — but confirm the env var on the live host before launch.

## P1 — Correctness, abuse, and scaling

### 4. X-Forwarded-For spoofing bypasses magic-link per-IP rate limit
- **Location:** `deploy/nginx/kbz.conf:121-130`
- **Evidence (verified):** The `/kbz/` location block sets `X-Real-IP` but **not** `X-Forwarded-For`. nginx `proxy_set_header` directives do not inherit from the server block once any is set in the location, so the server-level `X-Forwarded-For $proxy_add_x_forwarded_for` (line 92) does **not** apply inside `/kbz/`. `_client_ip()` trusts the leftmost XFF from a trusted peer, so a spoofed `X-Forwarded-For: 1.2.3.4` rotates the per-IP key and defeats the 30/hr magic-link limit (per-email 5/hr still applies).
- **Recommendation:** Add `proxy_set_header X-Forwarded-For $remote_addr;` to the `/kbz/` block, or switch `_client_ip()` to read `X-Real-IP`.

### 5. nginx does not strip `X-KBZ-Agent-Secret` on the public path
- **Location:** `deploy/nginx/kbz.conf:121-130`; `OPS.md:139-143`
- **Evidence:** Header-clearing directive documented in OPS.md is absent from the tracked config. Defense-in-depth: once the secret is set (per P0), a leaked secret could be replayed from the internet.
- **Recommendation:** Add `proxy_set_header X-KBZ-Agent-Secret "";` to the `/kbz/` block, ideally in the same change as setting the secret.

### 6. Stale support votes from thrown-out members survive and can push quorum
- **Location:** `src/kbz/services/member_service.py:72-134`
- **Evidence (verified):** `throw_out()` deactivates the member/bot (lines 95-125) but never deletes or decrements that user's `Support` rows. `member_count` drops on the next pulse while `support_count` retains ghost votes, so the lowered threshold can be met by a non-member's vote.
- **Recommendation:** In `throw_out()`, DELETE the user's Support rows and decrement `support_count` on affected in-flight proposals in the parent + child-action communities.

### 7. Membership escrow permanently orphaned when applicant amends
- **Location:** `src/kbz/services/proposal_service.py:1316-1320` (comment + direct cancel), `:1357-1416` (successor creation)
- **Evidence (verified):** The amend path comments "keep the escrow tied to the successor" and flips the original to CANCELED directly, but **no code re-keys the escrow Wallet's `owner_id` to the successor's new UUID.** `escrow_release(successor.id)` then finds nothing and the fee is locked forever.
- **Recommendation:** In `amend()`, `UPDATE Wallet SET owner_id = successor.id WHERE owner_kind = OWNER_ESCROW AND owner_id = original.id`, fulfilling the stated intent. Alternatively block amend for Membership proposals with a live escrow.

### 8. `invite_service.claim()` bypasses `MEMBERSHIP_GLOBAL_CAP`
- **Location:** `src/kbz/services/invite_service.py:104-250`
- **Evidence (verified):** `claim()` builds a Proposal via direct INSERT and replicates per-community dedupe but performs no global in-flight count; `MEMBERSHIP_GLOBAL_CAP` is not imported. N invite links → N concurrent membership proposals platform-wide.
- **Recommendation:** Mirror the `ProposalService.create` cap check (proposal_service.py:779-804) in `claim()` before the INSERT, or delegate to `ProposalService.create`.

### 9. Single uvicorn worker shares 40-conn pool with sim loop; long proxy timeout
- **Location:** `agents/run_with_viewer.py:443-454`; `src/kbz/database.py` (no `pool_timeout`); `deploy/nginx/kbz.conf:129` (`proxy_read_timeout 300s`)
- **Evidence (verified):** Single worker; `proxy_read_timeout 300s` on `/kbz/`; pool defaults to 40 conns / 30s timeout.
- **Recommendation:** Set `pool_timeout=5` in `create_async_engine` (fail fast → 503 not 30s queue), drop `/kbz/` `proxy_read_timeout` to 10-15s, and verify the `KBZ_AGENTS_ENABLED=0` kill switch works before launch.

### 10. `GET /users/me/flags` has no pagination
- **Location:** `src/kbz/routers/flags.py:136-159`; `src/kbz/services/flag_service.py:372-386`
- **Evidence (verified):** Handler passes only `flagger_user_id`/`community_id`; no `limit`/`offset`. Inconsistent with notifications (200 cap) and comments (500 cap).
- **Recommendation:** Add `limit`/`offset` Query params threaded into `list_my_flags`.

### 11. `FundingRequestIn` / `PaymentRequestIn` pitch has no max_length
- **Location:** `src/kbz/routers/wallets.py:297, 388` (`pitch: str = ""`, written to `proposal_text` at 352, 449)
- **Evidence (verified):** Bare `str`, bypassing the `_TEXT_MAX = 10_000` enforced by `ProposalCreate`.
- **Recommendation:** `pitch: str = Field(default="", max_length=10_000)` on both models (and bound `amount`).

### 12. `list_proposal_versions` N+1 enrich + no depth cap
- **Location:** `src/kbz/routers/proposals.py:127`; `src/kbz/services/proposal_service.py:1459-1466`
- **Recommendation:** Replace the per-version `enrich_one` comprehension with a single `svc.enrich(chain, ...)`; add a ~50-hop depth cap on the chain walk.

### 13. TKG embed queue unbounded; `QueueFull` handler is dead code
- **Location:** `src/kbz/services/tkg_ingestor.py:80` (`asyncio.Queue()` → maxsize 0 = unlimited), `:337-340` (dead `except QueueFull`)
- **Recommendation:** `asyncio.Queue(maxsize=500)` so `put_nowait` actually raises and the warning fires; document that TKG nodes have no prune path.

### 14. Per-agent in-memory sets / `action_history` grow unbounded
- **Location:** `agents/agent.py:89-93, 298, 523`
- **Recommendation:** Cap `action_history` to ~200 (only sliced `[-6:]`/`[-20:]`); periodically prune `supported_proposals`/`commented_proposals` against the live active-proposal set.

### 15. Model/migration schema divergence (test DB lacks prod constraints)
- **Redundant `ix_users_email_unique` never dropped** — `m1n2o3p4q5r6` adds `ix_users_email_lower_unique` but the upgrade never drops the old case-sensitive index; both coexist in prod, doubling write cost on `users.email`. **Add a migration: `DROP INDEX IF EXISTS ix_users_email_unique`.** (`alembic/versions/e0c1d2e3f4a5_human_auth.py:45`, `m1n2o3p4q5r6_users_email_unique.py`)
- **`AuthToken` / `Invite` missing `__table_args__`** — unique on `token_hash` and `invite_code` exist only in migration, absent from `create_all` test DB (security-relevant: token-hash uniqueness untested). (`src/kbz/models/auth.py:21,58`)
- **`BotProfile` missing `__table_args__`** — `(user_id, community_id)` unique exists only in migration; router SELECT-before-INSERT is not race-safe without it. (`src/kbz/models/bot_profile.py:22`)
- **Recommendation:** Add `__table_args__` to all three models mirroring the migrations so `create_all` and Alembic converge.

### 16. Test gaps (orchestrator pause/resume, closeness gate as logged-in non-member)
- Orchestrator `start_paused`/`auto_pause` (`agents/orchestrator.py:68-114, 605-619, 844-863`) have zero tests. The closeness 403 gate (`src/kbz/routers/closeness.py:38-43`) is tested only anonymously, not as a logged-in non-member — the regression-prone path. **Add targeted unit tests for both.**

## P2 — Defense-in-depth, hygiene, lower-priority tests

- **No nginx rate limiting on `/kbz/`** — add `limit_req_zone`/`limit_req` per-IP (e.g. 30r/s burst=60); fix `/healthz` 404 referenced in OPS rollback. (`deploy/nginx/kbz.conf`)
- **`work_tree` endpoint** (`src/kbz/routers/artifacts.py:457-506`) — anonymous, N+1 child-container query loop, no visibility gate, no community-existence 404. *Reclassified from P0 to P2:* it leaks artifact titles of private communities but no auth-token/financial data, and the N+1 is bounded by container count. Add a visibility gate + batch the child-container query.
- **Accepted ThrowOut doesn't cancel expelled member's in-flight proposals** — `_exec_throw_out` (`src/kbz/services/execution_service.py:57-60`) only calls `throw_out`; mirror the EndAction orphan-sweep (`:281-346`) to cancel the user's DRAFT/OUT_THERE/ON_THE_AIR proposals.
- **`highlights` N+1 `_root_id_of`** — same root cause as P0 #2; folded into that fix. Add the partial index `CREATE INDEX ... ON proposals (decided_at DESC) WHERE proposal_status='Accepted'`.
- **`LedgerEntry` indexes only in migration** — add the three to `__table_args__` for test/prod plan parity (no correctness impact). (`src/kbz/models/wallet.py:64-74`)
- **Turn-pacer `_reset_pacer` fixture** doesn't reset `_turn_pacer_lock` — latent deadlock risk for future error-injecting tests. (`tests/test_turn_pacer.py:19-26`)
- **Untested success paths:** `GET /admin/contact` inbox read, simulation API control endpoints with a live mock orchestrator. Add coverage.
